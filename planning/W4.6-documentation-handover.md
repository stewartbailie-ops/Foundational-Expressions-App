## 1. What Is Advisory Connect?

Advisory Connect is a SaaS platform for South African financial advisors. Each advisor gets a branded public profile page that captures leads (callback requests, referrals, will enquiries), displays financial content (news, exchange rates, infographic facts, calculators), and links to their financial platforms. Advisors manage their leads and profile settings through a self-service panel. The business owner manages all advisors through a master admin dashboard.

**Live URL:** `https://app.advisoryconnect.pro`  
**Source code:** `https://github.com/stewartbailie-ops/Advisory-Connect-Control-Panel`  
**Hosting:** Replit (auto-deploys when changes are pushed to the `main` branch on GitHub)

```
│            ┌───────────┴───────────┐                │

└─────────────────────────────────────────────────────┘

         │                          │ triggers redeploy
    Browser / PWA               GitHub repo
```

**Key architectural decisions:**
- Single server handles both the API and serves the React SPA — no separate frontend server
- In development, Vite runs as middleware inside Express (HMR works)
- In production, Vite output is compiled to `dist/public/` and served statically
- `.replit.app` URLs are 301-redirected to `advisoryconnect.pro` to keep one canonical domain
- Social media crawlers (WhatsApp, LinkedIn, etc.) receive server-rendered OG tags instead of the React SPA

## 5. Authentication

There are two separate auth systems running in parallel.

### 5.1 Master Admin

- **Login:** `POST /api/auth/login` — checks `ADMIN_EMAIL` + `ADMIN_PASSWORD` env vars
- **Session flag:** `session.authenticated = true`
- **Access:** Full read/write access to everything — all advisors, all leads, all settings
- **Default email:** `info@corefinancials.org` (overridden by `ADMIN_EMAIL` env var)

### 5.2 Advisor Self-Service (OTP)

- **Step 1:** `POST /api/advisor-auth/send-otp/:slug` — generates a 6-digit OTP, emails it to the advisor's registered email via SendGrid, stores it server-side for 10 minutes
- **Step 2:** `POST /api/advisor-auth/verify-otp/:slug` — validates the OTP, sets `session[advisor_{slug}] = true`
- **Access:** Can only read/write their own data (enforced by `canAccessAdvisor()` in routes)
- **Rate limiting:** 10 attempts per 15 minutes on all auth endpoints

### 5.3 Auth Middleware (`server/auth.ts`)

`requireAuth` runs on every request. It:
1. Passes through all non-API routes (React SPA handles those)
2. Passes through routes in `PUBLIC_API_ROUTES` (no auth needed)
3. Passes through advisor slug routes (public profile data)
4. Passes through advisor-owned profile routes (handler enforces ownership)
5. Checks `session.authenticated` for everything else
6. Returns `401 Unauthorized` if none of the above match

## 10. Lead Grader 2.0

Every lead is automatically scored when it arrives. Defined in `shared/schema.ts`.

**Score = sum of 5 categories (max 100):**

| Category | Max points | Criteria |
|---|---|---|
| Income | 35 | Upper bound of stated income bracket |
| Age | 20 | Sweet spot 35–55 (full 20pts), working life 28–65 (14pts) |
| Lifestyle complexity | 20 | +5 each: married, children, vehicle, property |
| Services interest | 15 | +4 per service mentioned, bonus for estate/wills/retirement |
| Source quality | 10 | Callback = 10, Will = 7, Referral = 5 |

**Grade thresholds:** Gold ≥75 · Silver ≥55 · Bronze ≥35 · Development <35

**Temperature** (separate axis, reflects intent/urgency):
- Hot = callback request
- Warm = will request, or referral with income/age data
- Cold = minimal data

## 11. Profile Section System

Each advisor profile is divided into named sections. The order is customisable per advisor via drag-and-drop in the admin panel. Stored as a JSON string in `advisors.profileSectionOrder`.

**Default section order:**

```
bio → moneyweb → secondnews → forex → interactive → funfacts →
individual → corporate → socials → callback → referral →
will → tools → platforms
```

Each section also has a `show*` boolean flag. If the flag is false, the section is hidden regardless of its order position.

## 14. Deployment (Replit)

1. Push to the `main` branch on GitHub
2. Replit detects the push and automatically rebuilds and republishes
3. The build runs `tsx script/build.ts` → esbuild bundles the server → Vite bundles the client
4. Static assets (including `client/public/facts/` images) are copied to `dist/public/`
5. The server starts with `NODE_ENV=production node dist/index.cjs`

**Redirect rule:** Any request to `*.replit.app` is 301-redirected to `advisoryconnect.pro` — this keeps URLs consistent and protects SEO.

## 15. Security Notes

- All sessions are `httpOnly`, `secure` (in production), `sameSite: lax`
- Helmet sets restrictive CSP headers — third-party scripts are limited to Google (reCAPTCHA)
- Rate limiting on all auth and OTP endpoints (10 req / 15 min)
- URL fields are validated server-side — only `http://` and `https://` prefixes accepted
- Zoho and SendGrid webhooks are verified using `crypto.timingSafeEqual` (timing-safe comparison)
- Image uploads: JPEG, PNG, WebP only, max 5MB. PDFs max 10MB.
- reCAPTCHA v2 on all public lead capture forms (bypassed gracefully if key not configured)

**Known limitation:** `ADMIN_PASSWORD` is used as both the admin credential and the session secret. These should ideally be separate values. A dedicated `SESSION_SECRET` env var is recommended for a future hardening pass.
